# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any skill, please report it responsibly:

1. **Do NOT** open a public issue
2. Email security concerns to the project maintainers
3. Include a description of the vulnerability, steps to reproduce, and potential impact

## Security Standards for Skills

All skills in this repository must follow these security standards:

- No hardcoded secrets, API keys, or credentials
- No internal URLs, account IDs, or infrastructure details
- No company-specific business information
- Examples must use generic placeholders (`example.com`, `<your-value>`)
- Credential references must use environment variables, not inline values

## Review Process

Every skill contribution is reviewed against the security checklist in [CONTRIBUTING.md](CONTRIBUTING.md) before being accepted.
